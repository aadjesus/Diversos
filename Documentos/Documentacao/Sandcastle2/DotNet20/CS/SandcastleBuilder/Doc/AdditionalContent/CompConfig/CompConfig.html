Custom Build Components

Custom build components can be used to add additional features to the help topics as they are generated by the Sandcastle BuildAssembler.exe tool and are supported by the help file builder. The following build components are supplied with the help file builder and are pre-configured for you:
  • The Show Missing Documentation Component
  • The Version Information Component
  • The Code Block Component
  • The Post-Transform Component
Third-party build components can also be added by inserting the component configuration data into the sandcastle.config file. It can be found in the Templates\ folder under the installation folder. Refer to the component's documentation for information on where to insert the component configuration.

If written to support it, the help file builder will also allow you to interactively configure the third-party build components and store the modified settings in the project file. This is accomplished using the project's ComponentConfigurations property. Select it and click the "..." button to the right of the property. A new dialog box will appear listing the components that allow configuration. Select the one you want and click the Configure button. A custom dialog box will appear that lets you modify the component's default settings. Refer to the component documentation for details on the available settings.

For information on writing your own custom build components, see the Code Project article Creating Custom Build Components for Sandcastle[^].
